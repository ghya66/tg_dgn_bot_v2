#### 💳 支付充值

## 📱 使用方式

### 命令入口
```
/help - 打开帮助中心
```

### 交互流程
1. 发送 `/help` 命令
2. 看到帮助中心主菜单
3. 点击感兴趣的分类按钮
4. 阅读详细帮助内容
5. 使用"返回"按钮导航
6. 使用"返回主菜单"回到 Bot 主界面

### 导航按钮
- **◀️ 返回帮助菜单** - 从分类页返回主菜单
- **🏠 返回主菜单** - 返回 Bot 主界面
- **/cancel** - 退出帮助系统

## 🛠️ 技术实现

### 模块结构
```
src/help/
├── __init__.py       # 模块导出
├── handler.py        # 处理器逻辑
└── content.py        # 帮助内容配置
```

### 核心组件

#### 1. ConversationHandler
使用 `ConversationHandler` 管理对话状态：
- **entry_points**: `/help` 命令
- **states**: `SHOWING_HELP` 状态处理所有回调
- **fallbacks**: `/cancel` 命令退出

#### 2. 回调路由
- `help_basic` - 基础功能
- `help_payment` - 支付充值
- `help_services` - 服务使用
- `help_query` - 查询功能
- `help_faq` - 常见问题
- `help_quick` - 快速开始
- `help_back` - 返回帮助菜单
- `back_to_main` - 返回主菜单

#### 3. 内容管理
所有帮助内容集中在 `content.py` 中：
- `MAIN_HELP_TEXT` - 主菜单文本
- `BASIC_HELP` - 基础功能帮助
- `PAYMENT_HELP` - 支付充值帮助
- `SERVICES_HELP` - 服务使用帮助
- `QUERY_HELP` - 查询功能帮助
- `FAQ_CONTENT` - FAQ 内容
- `QUICK_START` - 快速开始

## 🔧 维护和更新

### 更新帮助内容
编辑 `src/help/content.py` 文件：

```python
# 例如：更新 FAQ 内容
FAQ_CONTENT = """
❓ <b>常见问题 FAQ</b>

<b>Q1: 新问题标题？</b>
A: 新问题的回答...

...
"""
```

### 添加新分类
1. 在 `content.py` 中添加新内容常量
2. 在 `handler.py` 的 `show_help_category()` 中添加映射
3. 在主菜单键盘中添加新按钮
4. 更新回调路由模式

### 国际化支持
目前使用中文。如需支持多语言：
1. 为每种语言创建独立的内容文件
2. 根据用户语言设置加载对应内容
3. 使用 i18n 库管理翻译
